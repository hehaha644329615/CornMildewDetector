### 分析与总结

**数据分析**：
压根没有考虑拍照距离和透视角度，还错误的先增强数据再划分了，导致YOLO和FasterRCNN训练出来的精度大差不差的都很低，调了几次参数都没多大改善，开始下沉到框架训练的底层逐一排查，通过可视化验证集上的真实框和模型画的预测框（你来实现代码，反正是画几张图做readme文档的论据支撑，不用很严谨），发现数据不严谨，真的很影响模型的学习精度。发现了验证集的图在训练集上有，造成了数据泄露，重大踩坑了。还发现了最大的噪声是背景，有的图片在水泥地上拍，有的把蛇皮袋打开后拍，五花八门。因为霉点是黑的，采集的图像有些玉米粒缝隙是水泥地是黑的，玉米堆叠的阴影也是黑的，有些图片强光下，玉米粒的缝隙地面或者是蛇皮袋上的图案是白的，玉米粒的尾也是白的，模型很容易把霉点或者玉米粒的尾巴判定为背景。
**复盘总结**：
- 数据增强与train、val的划分顺序非常重要，处理不当造成严重的数据泄露；
- 背景照片的收集也很重要，训练集没有干净的背景图片，霉变样本（黑色）很容易误判为背景；
**v2优化方向**：
- 需要在普通手机上实时推理（ios / 鸿蒙）
- 模型大小需控制在 5MB 以内
- 收购现场网络不稳定，必须离线运行
